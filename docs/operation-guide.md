# 교육 운영 가이드

> 교육 운영자(강사)를 위한 준비·운영·개선 가이드입니다. 이 문서만으로 교육을 준비하고 운영할 수 있도록 구성했습니다.

---

## 1. 사전설문 설계

교육 효과를 높이려면 참가자의 현재 수준과 니즈를 미리 파악해야 합니다. 설문 결과를 기반으로 **직군별·문제유형별 그룹핑**을 진행합니다.

### 설문 항목 예시

#### 기본 정보

| 항목 | 예시 선택지 |
|------|------------|
| 소속 부서/직군 | 영업, 마케팅, 인사, 재무, 기획, 기타 |
| 현재 AI 도구 사용 경험 | 사용 안 함 / 가끔 사용 / 매일 사용 |
| 주로 사용하는 AI 도구 | Claude, ChatGPT, Gemini, Copilot, 기타, 없음 |

#### 업무 패턴 파악

| 항목 | 예시 선택지 |
|------|------------|
| 반복적으로 수행하는 업무 | 보고서 작성, 데이터 정리, 이메일 작성, 자료 요약, 기타 (서술) |
| 자동화하고 싶은 업무 1순위 | 서술형 |
| 업무에서 가장 시간이 오래 걸리는 작업 | 서술형 |

#### 교육 기대

| 항목 | 예시 선택지 |
|------|------------|
| 이 교육에서 얻고 싶은 것 | 실습 경험 / 활용 아이디어 / 보안 이해 / 기타 |
| AI 활용에 대한 우려 사항 | 보안 / 정확성 / 업무 적용 어려움 / 없음 |

### 그룹핑 기준

설문 결과를 바탕으로 다음과 같이 그룹을 나눕니다.

- **직군별**: 유사한 업무를 하는 사람끼리 묶어 실습 주제를 맞춤 제공
- **경험별**: AI 경험이 없는 그룹은 기초부터, 경험이 있는 그룹은 심화 실습 진행
- **문제유형별**: "보고서 자동화" 니즈와 "데이터 정리" 니즈를 구분하여 실습 유형 배정

## 2. 교육 당일 운영 체크리스트

### 교육 전 (D-1 ~ 당일 시작 전)

- [ ] 실습 환경 접속 확인 (AI 도구 로그인, 네트워크 등)
- [ ] 참가자 그룹핑 결과 확정 및 조 편성
- [ ] 실습용 샘플 데이터 준비 (개인정보 비식별 처리 완료)
- [ ] 프로젝터/화면 공유 테스트
- [ ] 보안 가이드 인쇄물 또는 링크 준비

### 교육 진행 중

- [ ] 보안 가이드 안내 (교육 시작 시 5분)
- [ ] 실습 목표 안내: "오늘 1개의 결과물을 만들어 갑니다"
- [ ] 그룹별 실습 시작 → 강사 순회 지원
- [ ] 중간 체크: 참가자별 진행 상황 확인 (막히는 사람 우선 지원)
- [ ] 결과물 공유 시간 (선택): 잘 만든 사례 2~3개 전체 공유

### 교육 후 (당일)

- [ ] 교육 후 설문 배포 (만족도 + 개선점)
- [ ] 참가자 결과물 저장/백업 안내
- [ ] 다음 단계 안내 (추가 학습 자료, 질문 채널 등)

## 3. 효과측정

### 측정 시점

교육 직후 만족도 조사만으로는 실제 효과를 알 수 없습니다. **1개월 후 사용률 추적**이 핵심입니다.

| 시점 | 측정 항목 | 방법 |
|------|----------|------|
| 교육 직후 | 만족도, 이해도, 결과물 완성 여부 | 설문 |
| 1주 후 | 결과물 활용 시도 여부 | 간단한 체크인 (메일/메신저) |
| 1개월 후 | 지속 사용 여부, 업무 적용 사례 | 설문 + 인터뷰 |

### 판단 기준

| 지표 | 목표 | 판단 |
|------|------|------|
| 교육 직후 결과물 완성률 | 80% 이상 | 실습 난이도 적정 |
| 1개월 후 주 1회 이상 사용률 | 30% 이상 | 교육 효과 있음 |
| 1개월 후 업무 적용 사례 보고 | 참가자의 20% 이상 | 실질적 가치 창출 |

!!! info "목표 미달 시"
    목표에 미달하면 아래 "개선 프로세스"를 통해 원인을 분석하고 다음 교육에 반영합니다.

## 4. 개선 프로세스

교육은 1회성이 아닌 **반복 개선 사이클**로 운영합니다.

### 개선 사이클

```
교육 실시 → 직후 설문 → 1개월 후 추적 → 원인 분석 → 개선 반영 → 다음 교육
```

### 직후 설문에서 수집할 개선 포인트

- 실습 난이도가 적절했는지 (너무 쉬움 / 적절 / 너무 어려움)
- 실습 시간이 충분했는지
- 강사 지원이 적시에 이루어졌는지
- 보안 가이드가 이해하기 쉬웠는지
- 추가로 배우고 싶은 주제

### 1개월 후 추적에서 파악할 것

- 사용하지 않게 된 이유 (잊어버림 / 필요 없음 / 어려움 / 보안 우려 등)
- 실제 업무에 적용한 사례와 효과
- 추가 지원이 필요한 부분

## 5. 과제 난이도 파일럿 테스트

새로운 실습 과제를 도입할 때는 **본 교육 전에 소규모 파일럿 테스트**를 진행합니다.

### 파일럿 절차

1. **대상 선정**: 본 교육 대상과 유사한 배경의 3~5명 선정
2. **실습 진행**: 동일한 환경에서 실습 과제 수행
3. **관찰 및 기록**:
    - 완료까지 소요 시간
    - 막히는 지점과 질문 내용
    - 결과물의 품질 수준
4. **난이도 판정**:

| 소요 시간 | 완료율 | 판정 | 조치 |
|----------|--------|------|------|
| 목표 시간의 50% 이내 | 100% | 너무 쉬움 | 심화 단계 추가 |
| 목표 시간의 80~120% | 80% 이상 | 적정 | 유지 |
| 목표 시간 초과 | 60% 미만 | 너무 어려움 | 단계 분리 또는 힌트 추가 |

## 6. 운영 부하 관리

### 원칙: 작게 시작하고 점진적으로 확장

한 번에 모든 대상·모든 유형의 교육을 진행하면 운영 부하가 과도해집니다.

### 단계별 확장 전략

| 단계 | 대상 | 실습 유형 수 | 비고 |
|------|------|-------------|------|
| 1단계 | 1개 부서 파일럿 | 2~3개 | 핵심 유형만 검증 |
| 2단계 | 전사 1차 교육 | 2~3개 | 파일럿 결과 반영 |
| 3단계 | 전사 확대 | 유형 추가 | 수요 기반 확장 |

### 운영 부하 관리 팁

- **초반에는 실습 유형을 2~3개로 제한**: 강사가 충분히 숙달된 유형만 운영
- **강사 1인당 참가자 수 제한**: 실습 지원을 위해 강사 1인당 10~15명 권장
- **자주 묻는 질문(FAQ) 정리**: 반복 질문은 문서화하여 자기 해결 유도
- **교육 운영 체크리스트 재사용**: 매 회차마다 동일한 체크리스트를 사용하여 누락 방지
