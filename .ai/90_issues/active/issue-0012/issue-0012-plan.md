# Issue #12 실행계획 발표 슬라이드에 "보안 및 개인정보 가이드" 섹션 추가

---

## 결정사항

### 삽입 위치·분리 여부 (Task 1 결과)

- **선택**: 단일 `slides.md` 내 본 흐름 끝(How 직후) · 감사 직전 / 독립 `layout: section` 섹션으로 분리
- **분리 방식**: 별도 Slidev 라우트(B안) 채택하지 않음 — 단일 파일 내 섹션 분리
- **흐름**: `... → How → [신규] 보안 및 개인정보 가이드 (section) → 감사`

**근거**

1. **docs 구조 정렬**: `mkdocs.yml` nav에서 `security-guide.md`는 본 콘텐츠 흐름과 나란히 놓인 독립 페이지(`홈 / 보안 및 개인정보 가이드 / 교육 운영 가이드`). 슬라이드에서도 동일 위상으로 본 흐름(Why/Who/What/How) 옆 독립 섹션에 두어야 구조가 정렬됨
2. **교육 흐름 보존**: 본 흐름(Why/Who/What/How)은 교육 소개·실습 가이드의 일관된 메시지. 본 흐름 중간(Who 다음, What 다음 등)에 보안을 끼워넣으면 흐름이 끊김
3. **마무리 강조**: 본 흐름이 끝난 뒤 보안 섹션을 두면 발표 마무리에 안전 사용 메시지를 남길 수 있음
4. **운영 복잡도**: 별도 Slidev 라우트는 ADR-0003(자체 슬라이드 모드 제거 → 단순화)의 결을 거스름. 단일 파일 내 섹션 분리로 충분
