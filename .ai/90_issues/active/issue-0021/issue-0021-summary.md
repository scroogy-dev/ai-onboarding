    11. **WHO → WHAT 이동: "AI 활용 3단계와 내 위치" + 1↔2단계 grid + 결정 룰 grid + ②번 초점 인용 + "왜 단계가 이어지나요?" tip** — 학습자 분류가 아니라 "교육이 다루는 범위·산출물의 단계"라 WHAT에 더 맞음. WHAT 위계: 교육 목표 → 3단계 모델 → 결과물 트랙 표 (자연 매핑). `{ #stage-model }` ID는 H3에 그대로 박혀 있어 operation-guide.md 링크 정상
    12. **결정 룰 grid 슬림화** — N-2 사다리와의 메시지 중복 정리(옵션 1 채택): 카드의 "**특성**" 줄 제거 (`매번 결과가 다를 수 있음` / `동일한 품질로 반복 보장`은 N-2 [3]/[4]가 결과형으로 이미 다룸), "본 교육 ②번 쪽 초점" 인용도 N-2 [4] 결론과 시멘틱 중복으로 제거. 결정 룰은 라벨 + **적합** 한 줄만 남겨 "언제 ①/언제 ②" 실용 분기 가이드로 역할 차별화
    13. H3 제목 다듬기: "왜 지금, 당신이 AI를 배워야 할까요?" → "AI를 배워야 하는 이유" / "대상자 프로필" → "교육 대상" (slides 본문과도 일관)
    14. WHY 내 H3 swap: `매번 시키기 vs 소프트웨어로 만들기` ↔ `소프트웨어가 만드는 가치` — W1을 마지막에 두어 "AI가 등장하면서 이 가치를 만드는 길이 비개발자에게도 열렸습니다" 한 줄이 자연스러운 닫는 메시지로 작동
    15. **어휘 통일** (옵션 A 채택): 학습자 노출 자료에서 "강의" → "교육"으로 일괄 통일. docs 3건(`이 강의가` `본 강의가` `이론 강의 → 이론 학습`) + slides 3건(L73·L403 발표자 노트의 "본 강의·90분 강의" → "교육" / L572 본문 "이론 강의" → "이론 학습"). ADR-0005 본문·이슈 노트·아카이브는 의사결정 기록의 시점 어휘로 보존
    16. **N-2 (매번 시키기 vs 소프트웨어로 만들기) WHY → WHAT 첫 H3로 이동** (옵션 B 채택). WHY는 동기 부여(3축 + W1 거시 배경)에 집중, WHAT은 "본 교육 접근 정의(N-2) → 교육 목표 → 3단계 모델 → 결과물" 콘텐츠 톤 일관. 본질 메시지 못은 WHAT에 집중(N-2 [4]+W3 박스, 결정 룰 박스) + HOW(왜 계획? 박스) 분포 3곳 유지. ADR-0005 후속 변경 Note에 위치 변경 추가
    17. WHAT 안에서 H3 swap: `매번 시키기 vs 소프트웨어로 만들기` ↔ `교육 목표`. 새 흐름 = 교육 목표(1개 만든다) → 그 1개의 정체 정의(②번 = 소프트웨어) → 3단계 → 결과물. H2 직답이 즉시 손에 잡히도록 결론 우선 배치
    18. **결정 룰 박스를 N-2 H3 첫 부분으로 통합 + 비교 형식 강화**. (a) WHAT "AI 활용 3단계" H3에서 결정 룰 grid 제거, N-2 H3 도입 직후로 이동. (b) 라벨 강화: `① 직접 지시 (= 매번 시키기)` / `② 소프트웨어로 만들기 (본 교육이 강조하는 쪽)` — H3 제목과 직접 매핑. (c) 사다리 단계 표기를 ①②③④ → `[1][2][3][4]` 대괄호로 변경해 결정 룰 ①②와 인지 충돌 회피. 새 N-2 흐름: 분기 비교 grid → 다리 한 줄 → 사다리 풀이 → 핵심 박스. ADR-0005 후속 변경 Note에 위치 변경 추가
  - **ADR-0005 운용 미세 조정**: 결정 룰 박스 위치가 "WHO 섹션 W3-A 직후" → "WHAT 섹션 교육 목표 후"로 이동했음. 본질 메시지 못 분포 3곳(N-2 [4]+W3 박스 / 결정 룰 박스 / 왜 계획? 박스)은 그대로 유지 — ADR 재정의 아님. **ADR-0005 본문 갱신 완료**: ① 영향 범위에 What 섹션 추가 ② 결정 §2 표 직후에 "후속 변경 (Issue #21, 2026-05-03)" Note 추가 (3개 위치 운용 변경: 도입부 못→N-2 [4] 수렴 / 핵심 인용구→admonition 박스 격상 / 결정 룰→What 섹션 이동) ③ 결과·후속 작업 항목에 위치 변경 메모 inline 추가
