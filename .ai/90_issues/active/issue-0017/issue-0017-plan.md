## Tasks

> 이 이슈는 사용자가 개선 항목을 한 번에 하나씩 메시지로 전달하면, AI가 그 자리에서 Task로 등록·수행·기록하는 **점진적(incremental) 방식**으로 진행합니다.
> 따라서 Task 목록은 이슈 시작 시점에는 비어 있고, 작업이 진행되면서 아래에 누적됩니다.

### 운영 규칙

- 사용자가 새 개선 항목 메시지를 보내면, AI는 다음을 수행한다.
  1. **Phase 1(`docs/`)** 또는 **Phase 2(`slides/`)** 중 어느 단계에 속하는지 판단해 해당 섹션에 새 Task를 추가한다 (각 Phase 안에서 Task 번호는 추가 순서대로 1, 2, 3 …).
  2. Task의 목표·작업 내용·완료 기준을 메시지에 근거해 채운다. 메시지에 결정 사항이 누락되어 있고 메모리·`AI-CONTEXT.md`로도 판단이 안 서면 임의 추측하지 않고 사용자에게 확인한다.
  3. 작업을 수행하고, 완료 시 체크박스를 체크한 뒤 `issue-0017-summary.md`를 갱신한다.
- `docs/`(SSoT) 변경 Task가 완료되면 AI는 자동으로 **Phase 2 — Task 1(슬라이드 동기화)** 의 "작업 내용"에 해당 변경분을 누적 기록한다. (별도 동기화 Task를 새로 만들지 않음 — Phase 2 Task 1은 이슈 시작 시 한 번 등록되어 이슈 전체 기간 동안 누적된다.) 슬라이드 영향이 없으면 "슬라이드 영향 없음"으로 메모하고 스킵한다.
- `slides/` 단독 개선 항목은 Phase 2 Task 2 이후에 직접 등록한다.
- **이슈 종료 트리거**: 사용자가 메시지로 명시적으로 "이슈 종료"를 알릴 때까지 점진적으로 누적한다. 종료 메시지 수신 시 `/issue-work` 완료 절차(Phase 2 Task 1 포함 모든 Task 체크 확인 → summary 갱신 → archive 이관)를 수행한다.
