# Issue #0001 스펙

## 목표 (Goal)

비개발자(임직원, 학생/일반인) 대상 AI 활용 교육자료를 MkDocs + Material 기반 정적 사이트와 컨플루언스 위키 양쪽으로 발행할 수 있는 구조를 구축하고, 핵심 교육 콘텐츠를 작성한다.

---

## 범위 (Scope)

**포함 (In)**

- MkDocs + Material 프로젝트 세팅 (mkdocs.yml, CI/CD, .gitignore 등)
- 메인 페이지(index.md): Why(소프트웨어 가치와 AI의 역할) + What(교육 목표) + How(실행 계획 개요)
- 보안 및 개인정보 가이드 페이지
- 교육 운영 가이드 페이지 (사전설문, 효과측정, 개선 프로세스)
- 컨플루언스 업로드 (Atlassian MCP를 통해 위키 페이지로 게시)

**비포함 (Out)**

- How 하위 실습 페이지 4종 (실습 내용 미확정 → 별도 이슈로 진행)
  - 비개발자 직장인 실습 - 유형 A (반복 보고서 자동화)
  - 비개발자 직장인 실습 - 유형 B (데이터 정리/변환)
  - 학생/일반인 실습 - Gems 활용 (오답노트, 출제기)
  - 학생/일반인 실습 - Cowork 활용 (엑셀 연동)
- 개발자 대상 교육자료 (클로드 코드 + MCP 연동은 별도 이슈)
- 사전설문 폼 자체의 제작 (가이드 내 설문 항목 예시만 포함)
- MCP 연동 샘플 코드 작성
- scripts/sync-confluence.py 실제 구현 (스켈레톤만 생성)

---

## 아키텍처

- **SSoT**: `docs/` 디렉토리의 마크다운 파일
- **정적 사이트**: MkDocs + Material → GitHub Pages (비개발자 접근용)
- **위키**: Atlassian MCP → 컨플루언스 (강사용)
- **브랜치 전략**: main(소스) / gh-pages(빌드 결과물, 자동 생성)

---

## 완료의 정의 (Definition of Done)

- [ ] MkDocs + Material 프로젝트가 로컬에서 빌드·서빙 가능하다
- [ ] GitHub Actions CI가 main push 시 gh-pages로 배포하도록 설정되어 있다
- [ ] 메인 페이지(index.md)에 Why/What/How 섹션이 모두 채워져 있다
- [ ] 보안 및 개인정보 가이드 페이지가 존재한다
- [ ] 교육 운영 가이드 페이지가 존재하고, 사전설문 항목/효과측정 기준/개선 프로세스가 포함되어 있다
- [ ] 모든 페이지의 톤이 비개발자가 읽어도 이해할 수 있는 평이한 수준이다
- [ ] 모든 페이지가 컨플루언스에 업로드되어 있다
