- **특이 사항**:

### Task 4: 로컬 렌더링 및 자체 리뷰

- **결과**: 완료
- **수행 내용 요약**: `python3 -m mkdocs build --strict`로 로컬 빌드 검증. 초회 빌드에서 한글 heading의 slugify 이슈로 2건의 앵커 깨짐 발견 → 수정 후 strict 빌드 통과. DoD 9개 항목 전체 대조 완료.
- **특이 사항**:
  - **slugify 한글 처리 문제**: MkDocs 기본 slugify가 한글을 제거하여 "## Who — 누구를 위한 교육인가요?"는 `#who`로, "### 준비사항"은 `_N` 형태의 불안정한 anchor로 생성됨.
  - **해결책**: 전역 slugify 설정 변경(mkdocs.yml)은 blast radius가 커서 보류. 대신 이번 이슈에 필요한 "### 준비사항" 헤딩에 attr_list `{ #preparation }`로 명시 anchor 부여. Who 섹션은 기본 `#who`가 그대로 동작.
  - **향후 과제**: 다른 문서에도 한글 헤딩 deep link가 늘어나면 mkdocs.yml의 toc slugify를 pymdownx.slugs.slugify로 교체하는 것이 근본 해결책. 별도 이슈로 떼어낼 가치 있음.
  - **audit 파일 처리**: `.ai/99_workspace/issue-0005-audit.md`는 작업 과정의 중간 산출물이지만 충돌 지점·범위 외 항목(C4, C5)이 후속 이슈 후보로 기록되어 있어 **남겨둠**. 사용자 판단이 필요하면 삭제 여부 논의 가능.
  - **DoD 대조 결과**: 9개 항목 모두 충족.
