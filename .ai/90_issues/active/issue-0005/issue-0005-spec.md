# Issue #0005 스펙 — 교육자료 보강 (대상자·사전지식·준비사항)

> GitHub: https://github.com/scroogy-dev/ai-onboarding/issues/5

## 목표 (Goal)

교육자료에 **Who(대상자) / 사전지식 / 준비사항** 섹션을 추가하여, 참가자와 운영자 모두가 교육 시작 전에 자신이 이 교육의 대상인지, 무엇을 준비해야 하는지 판단할 수 있도록 한다. 기존 Why → What → How 흐름에 **Who**를 Why와 What 사이에 삽입하여 "왜 배우는지 → 누가 대상인지 → 무엇을 얻는지 → 어떻게 진행되는지" 순서로 읽히게 한다.

---

## 범위 (Scope)

**포함 (In)**

- `docs/index.md`에 **Who(대상자) / 사전지식 / 준비사항** 섹션 신설 (위치: Why 뒤, What 앞)
- Who(대상자) 정의: 기본적인 컴퓨터 조작이 가능하고, **ChatGPT·Gemini·Claude 등 챗봇형 AI를 한 번이라도 써 본 사람**을 전제로 명시
- 사전지식: 요구되는 최소 수준과 **요구하지 않는 것**을 두 그룹으로 명확히 분리한 **불릿 리스트**로 명시 (표는 행 단위 페어링 오해를 유발하므로 사용하지 않음)
  - 요구함: 기본 컴퓨터 조작(파일 업로드/다운로드, 웹 브라우저 사용), **기본적인 웹 검색**, 챗봇 AI와 짧은 대화 경험
  - 요구하지 않음: 프로그래밍 / 코딩 지식(API 사용 포함), 프롬프트 엔지니어링 이론, 특정 AI 도구의 고급 기능 숙련도
- 준비사항: 참가자가 교육 당일까지 갖춰야 할 것(계정, 기기, 네트워크, 샘플 데이터 등)을 체크리스트 형태로 제공
- 임직원 트랙 / 학생·일반인 트랙의 차이가 있으면 표로 분리 표기
- `docs/operation-guide.md`의 사전 체크리스트와 내용이 어긋나지 않도록 교차 검토
- (작업 중 추가) `docs/security-guide.md`의 "비식별 처리" 용어를 평이한 표현("개인정보 가리고 입력하기")으로 정리하고, 임직원·학생 양 트랙에 동일하게 적용

**비포함 (Out)**

- 실습 페이지 신규 작성 (별도 이슈)
- 대상자 자체를 바꾸거나 추가하는 의사결정 (현 index.md의 "직장인/학생·일반인" 구도는 유지)
- 저작권·허위정보 등 security-guide의 **신규 보강** (별도 이슈) — 단, 기존 항목의 용어 일관성 정리는 본 이슈에 포함됨
- mkdocs nav 구조 변경
