# Issue #17 실행계획 — 교육 자료 개선

> 스펙: [issue-0017-spec.md](./issue-0017-spec.md)

---

## Tasks

> 이 이슈는 사용자가 개선 항목을 한 번에 하나씩 메시지로 전달하면, AI가 그 자리에서 Task로 등록·수행·기록하는 **점진적(incremental) 방식**으로 진행합니다.
> 따라서 Task 목록은 이슈 시작 시점에는 비어 있고, 작업이 진행되면서 아래에 누적됩니다.

### 운영 규칙

- 사용자가 새 개선 항목 메시지를 보내면, AI는 다음을 수행한다.
  1. **Phase 1(`docs/`)** 또는 **Phase 2(`slides/`)** 중 어느 단계에 속하는지 판단해 해당 섹션에 새 Task를 추가한다 (각 Phase 안에서 Task 번호는 추가 순서대로 1, 2, 3 …).
  2. Task의 목표·작업 내용·완료 기준을 메시지에 근거해 채운다. 메시지에 결정 사항이 누락되어 있고 메모리·`AI-CONTEXT.md`로도 판단이 안 서면 임의 추측하지 않고 사용자에게 확인한다.
  3. 작업을 수행하고, 완료 시 체크박스를 체크한 뒤 `issue-0017-summary.md`를 갱신한다.
- `docs/`(SSoT) 변경 Task가 완료되면 AI는 자동으로 **Phase 2 — Task 1(슬라이드 동기화)** 의 "작업 내용"에 해당 변경분을 누적 기록한다. (별도 동기화 Task를 새로 만들지 않음 — Phase 2 Task 1은 이슈 시작 시 한 번 등록되어 이슈 전체 기간 동안 누적된다.) 슬라이드 영향이 없으면 "슬라이드 영향 없음"으로 메모하고 스킵한다.
- `slides/` 단독 개선 항목은 Phase 2 Task 2 이후에 직접 등록한다.
- **이슈 종료 트리거**: 사용자가 메시지로 명시적으로 "이슈 종료"를 알릴 때까지 점진적으로 누적한다. 종료 메시지 수신 시 `/issue-work` 완료 절차(Phase 2 Task 1 포함 모든 Task 체크 확인 → summary 갱신 → archive 이관)를 수행한다.

- **목표**: 모든 슬라이드에 발표 진행도를 보여주는 시각 장치를 추가한다 — 우하단 페이지 인디케이터 + 슬라이드 최하단을 가로지르는 진행 바(프로그레스 게이지).
- **작업 내용**:
  1. `slides/global-bottom.vue` 신규 생성 — Slidev convention에 따라 모든 슬라이드 하단에 자동 마운트되는 글로벌 레이어
  2. `useNav()` composable로 `currentSlideNo / total` 가져옴, `progress = (currentSlideNo / total) * 100` computed
  3. 두 요소 배치:
     - **우하단 페이지 인디케이터**: `absolute bottom-1.5 right-3 text-[10px] opacity-30 font-mono leading-none` — `{{ currentSlideNo }} / {{ total }}` (font-mono = 개발자/IDE 톤, 거의 안 보이는 보조 톤으로 톤 다운)
     - **최하단 진행 바**: `absolute bottom-0 left-0 right-0 h-0.5 bg-white/5` 트랙 + 안쪽 fill `bg-cyan-400/70` + `transition-all duration-300 ease-out`로 페이지 전환 시 부드럽게 늘어남. fill width는 `:style="{ width: progress + '%' }"`
  4. 진행 바 fill 색은 **cyan-400/70** — Task 9 mermaid 네온 팔레트(stage3 cyan-400)와 정렬해 슬라이드 시각 어휘 일관
  5. `print:hidden`로 PDF export 시 둘 다 자동 숨김
- **완료 기준**:
  - 모든 슬라이드 우하단에 페이지 번호 차분 표시
  - 슬라이드 최하단 가장자리에 가는 cyan 바가 페이지 진행에 맞춰 좌→우로 늘어남, 페이지 전환 시 부드러운 애니메이션
  - 슬라이드 콘텐츠 클릭 인터랙션 영향 없음 (인디케이터·바 모두 작은 영역, pointer-events 충돌 없음)
- **반복 이력**:
  1. 1차: 좌하단 페이지 + 우하단 ← → 버튼 풀-바 (A안 풀바) → 사용자 피드백 "화살표는 없애도, 우측 아래에 페이지 보여주면서 가장 하단에 바 같은데 늘어나는 형태도 가능?"
  2. 2차: ← → 버튼 제거 + 페이지 인디케이터 우하단 이동 + 최하단 진행 바 추가. 프로그레스 게이지 시각으로 진행도 표현 — 모던 슬라이드 표준(Linear/Notion 데모 톤)과 정합.
  3. 3차(현재): 페이지 인디케이터를 더 작게·흐리게 톤 다운 (`text-xs opacity-50` → `text-[10px] opacity-30`, `bottom-2 right-4` → `bottom-1.5 right-3`, `leading-none` 추가). 사용자 피드백 — 진행 바가 시각 메인이고 페이지 번호는 거의 안 보이는 보조로 두는 모던 톤이 더 적합.
- **시각 어휘 학습**: 발표 진행도는 "수치 표시"보다 "공간 게이지"가 인지 부담이 적다. 페이지 번호는 정확한 위치 확인용 보조이며, 청중·발표자 시각 메인은 진행 바. 정확한 위치 확인은 발표자 모드(`/presenter/`)에서 별도 처리.
