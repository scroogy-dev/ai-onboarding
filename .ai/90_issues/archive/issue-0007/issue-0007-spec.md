# Issue #7 스펙 교육 내용 개선 및 슬라이드 개선

> GitHub: https://github.com/scroogy-dev/ai-onboarding/issues/7

## 목표 (Goal)

비개발자 교육 효과를 높이기 위해 실습 외 교육 내용을 점검·보강하고, **AI 활용 3단계 모델(챗봇 → 프롬프트 재활용 → 에이전트)** 을 도입하여 학습 사다리를 명확히 한다. 또한 텍스트 위주의 슬라이드를 표·차트·그림 중심으로 재구성한다.

---

## 범위 (Scope)

**포함 (In)**

- 실습 섹션을 제외한 모든 교육 내용(개요, 이론, 활용 사례 등)의 가독성·정확성 점검 및 보강
- 실습 도구 범위 결정 (Claude만 사용할지, Gemini도 함께 다룰지) 및 단계별 도구 매핑
- **AI 활용 3단계 모델 도입** — 1) 챗봇 사용, 2) 프롬프트 재활용/정교화(Projects·Gems 등), 3) 에이전트 활용(Claude Cowork 등)
- **2단계·3단계 실습 분기 구조화** — 참가자 수준·목표에 따른 실습 경로 제시
- **대상자 재정의** — 학생 트랙에서 개발 진로 희망자 제외를 명시
- **사전설문의 활용 수준 선택지 재정비** — 3단계 모델에 맞춰 정렬
- 슬라이드/문서의 시각화 강화 (표, 차트, 다이어그램, 이미지 등)
- 텍스트 과다 구간을 시각 요소로 치환하여 내용 축약

**비포함 (Out)**

- 실습 시나리오 자체의 수정 (별도 이슈에서 다룸)
- MkDocs 테마/레이아웃의 대대적 변경
- 새로운 도구(Claude/Gemini 외) 추가
- **개발 진로 희망 학생 대상 콘텐츠** — 별도 개발 교육에서 다룸
- **Claude Code CLI 실습** — 3단계 개념 설명 시 Claude 데스크탑과 함께 언급만 하고, 본 교육에서는 실습하지 않음
- **2단계·3단계 실습 시나리오 상세 설계** — 2단계(Claude Projects)·3단계(Claude Cowork) 실습은 본 교육의 일부이나, 각 실습의 상세 시나리오·절차는 별도 이슈에서 다룸. 본 이슈에서는 **실습의 존재·단계 매핑·문서 구조**만 반영
- **FAQ 페이지 신설** — 본 이슈에서는 참조 placeholder만 두고, 별도 이슈에서 구축

## 완료의 정의 (Definition of Done)

- [ ] 실습 외 교육 내용 전체에 대한 점검 결과가 정리되어 있다
- [ ] 실습 도구 범위(Claude 단독 vs Claude+Gemini)에 대한 결정과 근거가 기록되어 있다 (필요 시 ADR)
- [ ] 결정된 실습 도구 범위가 관련 문서에 반영되어 있다
- [ ] **AI 활용 3단계 모델이 `docs/index.md`에 도입되어 있고, 참가자가 자신의 현재 단계를 인지할 수 있도록 기술되어 있다**
- [ ] **학생 트랙 대상에서 개발 진로 희망자 제외가 명시되어 있다**
- [ ] **사전설문(`docs/operation-guide.md`)의 "현재 AI 활용 수준" 선택지가 3단계 모델에 맞춰 재정비되어 있다**
- [ ] 텍스트 위주 구간이 표·차트·그림 등 시각 요소로 치환되거나 보강되어 있다
- [ ] 변경된 문서가 `mkdocs serve`로 정상 렌더링된다
