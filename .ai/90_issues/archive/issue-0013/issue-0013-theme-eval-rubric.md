## 1. 평가 기준 (6개)

각 항목은 후보별로 **A / B / C** 3등급으로 판정한다 (A=충족, B=일부 충족, C=미흡).

> 1.1~1.5는 이슈 본문에 명시된 기준, **1.6은 본 이슈에서 추가**된 기준이다 (사유: 2·3단계 실습 슬라이드에 코드 블록 도입이 예상되며, 코드 블록 시각 품질이 발표 톤을 좌우하는 비중이 큼).

### 1.1 가독성

비개발자 청중·발표 환경(노트북 화면 또는 빔프로젝터 16:9, 객석 후방 5m 이내) 가시성.

| 측정 항목 | 합격선 |
|----------|--------|
| 본문 한국어 글자 크기 (1080p 슬라이드 기준) | ≥ 28px (`text-xl`/`text-2xl` 상당) 가독 |
| 본문/배경 명도 대비 | WCAG 4.5:1 이상 |
| 행간 (line-height) | ≥ 1.5 |
| 어두운 발표장에서 눈부심 | 흰 배경 시 옵션으로 dim/dark 전환 가능 |

### 1.2 빌트인 레이아웃 다양성

현재 `slides/slides.md`에서 실사용 중인 5종 + 신규 후보를 어디까지 빌트인으로 커버하는지.

| 측정 항목 | 합격선 |
|----------|--------|
| 현행 사용 중 레이아웃 5종 (`cover`, `section`, `statement`, `center`, `end`) 빌트인 지원 | 5/5 |
| 추가로 사용 가치가 있는 레이아웃 (`two-cols`, `image-right`, `quote`, `fact`, `intro`) 빌트인 지원 개수 | ≥ 3/5 |
| 두 컬럼 패턴 (`<div grid="~ cols-2">`) 무리 없이 동작 | UnoCSS 또는 동등 유틸이 테마와 함께 제공 |

### 1.3 강조 · admonition 등가물 지원

이슈 #9에서 도입한 `border-l-4 border-{color}-400 ... bg-{color}-50/40` 패턴이 새 테마에서 어떻게 재현되는가.

| 측정 항목 | 합격선 |
|----------|--------|
| `border-l-4` UnoCSS 유틸이 그대로 동작 | A: 테마가 UnoCSS 그대로 노출 / B: 동등 유틸 제공 / C: 자체 컴포넌트 신설 필요 |
| 빌트인 admonition 컴포넌트 존재 (note/warning/danger 등) | 있으면 가산점 — 없어도 (a) 통과 시 합격 |
| 다크/라이트 모드에서 `bg-{color}-50/40` 가독성 유지 | 흰 텍스트 + 짙은 색 톤으로 대비 4.5:1 유지 |

### 1.4 한국어 폰트 매칭

| 측정 항목 | 합격선 |
|----------|--------|
| Sans/Serif/Mono 폰트 패밀리를 테마 변수로 오버라이드 가능 | `unocss.config` 또는 `style.css`에서 단일 지점 변경 |
| Noto Sans KR / Noto Serif KR fallback 자연스러움 (자형 흔들림·줄바꿈 깨짐 없음) | 한·영 혼합 본문에서 baseline 정렬 일치 |
| 코드 블록의 한글 inline (Pretendard/D2Coding 류) 균형 | 전각/반각 폭 차이로 인한 정렬 깨짐 없음 |

### 1.5 유지보수 활성도

| 측정 항목 | 합격선 |
|----------|--------|
| 최근 6개월 npm 릴리스 | ≥ 1회 |
| 마지막 GitHub 커밋 | 최근 12개월 이내 |
| Slidev `^52.x` 호환 명시 | package.json `peerDependencies` 또는 README에 명시 |
| 미해결 critical 이슈(렌더 깨짐·빌드 실패) | 0 |

### 1.6 코드 블록 시각 품질

발표 톤에 직결되는 항목. 현 `slides.md`에는 코드 블록이 없으나, 2·3단계 실습 슬라이드(프롬프트 예시, 터미널 출력 등)에 도입이 예상됨.

| 측정 항목 | 합격선 |
|----------|--------|
| 코드 블록 컨테이너 디자인 (윈도우 컨트롤·라운드 코너·그림자 등) | macOS Keynote/Apple 풍 정돈된 컨테이너 또는 그에 준하는 시각 마감 |
| Monospace 폰트 자형 | 한·영 혼합 시 baseline·폭 균형 (Pretendard/D2Coding/JetBrains Mono 등 fallback 가능) |
| Shiki highlighter 색 테마 | 라이트/다크 양쪽에서 가독성 4.5:1 유지, `highlighter: shiki` frontmatter 동작 |
| inline code (`` `…` ``) 시각 | 본문 흐름과 충돌 없는 배경·여백 |
| 슬라이드 본문 톤과 일관성 | 코드 블록만 톤이 튀지 않음 (배경·테두리가 전체 디자인 시스템에 정렬) |

> Task 2 진입 시 비교 샘플에 **코드 블록 1종**(예: 짧은 프롬프트 + 터미널 1줄)을 추가해 본 항목을 동등 조건에서 평가한다.

## 3. 변경 시 보존해야 할 자산 체크리스트

새 테마/커스터마이징 적용 시 아래 자산은 **시각적·의미적 변형 없이** 동일하게 노출되어야 한다.

### 3.1 ADR-0001 핵심 메시지 자산 (변형 금지)

| 자산 | 위치 | 검증 방법 |
|------|------|----------|
| "AI 활용 3단계 모델" 메시지 | `slides/slides.md` "AI 활용 3단계와 내 위치" 슬라이드 | 본문 텍스트 1자 단위 일치 |
| 단계별 명칭 (1단계 챗봇 사용 / 2단계 프롬프트 재활용 / 3단계 에이전트 활용) | 동 슬라이드 + "3단계 상세" 표 | 명칭 변경 금지 |
| "참가 전제 = 1단계 통과" 정의 | "AI 활용 3단계와 내 위치" 본문 | 의미 보존 |

### 3.2 ADR-0002 변형 금지 자산

| 자산 | 위치 (slides.md) | 호환성 검증 |
|------|------------------|------------|
| **Mermaid #1: 3단계 모델 flowchart** | "AI 활용 3단계와 내 위치" — `mermaid {scale: 0.85}` flowchart LR + `style A/B/C` | 새 테마에서 mermaid 렌더 + scale + node fill 색상 재현 |
| **Mermaid #2: 실습 접근법 flowchart** | "실습 접근법: 계획 → 실행" — `flowchart LR` 4노드 + 점선 피드백 화살표 | mermaid 렌더 + 점선(`-.->`) 동작 |
| **3단계 비교 표** | "3단계 상세" 슬라이드 — markdown table + inline `<style>{ font-size: 0.85em; }</style>` | 표 스타일 오버라이드 동작, `&nbsp;` 줄바꿈 제어 유지 |

### 3.3 이슈 #9 도입 강조 박스 (`border-l-4` UnoCSS 패턴)

| 색상 | 용처 | 슬라이드 |
|------|------|---------|
| **red** | 학생 진로 외 안내 ❌ + Claude Pro 필수 ⚠️ + 보안 핵심 원칙 | "학생 트랙 범위 안내", "준비사항 — Claude Pro 필수", "공통 원칙 — 트랙 무관 동일" |
| **orange** | 엔터프라이즈 한계 ⚠️ + 무료 AI 한계 | "임직원 — 엔터프라이즈 AI는 안전한가?", "학생·일반인 — 무료 AI 도구의 특성" |
| **blue** | 비식별 처리 예시 (정보형) | "임직원 — 그래도 지켜야 할 것" |

**호환성 분기**:
- (a) 새 테마가 UnoCSS 그대로 노출 → **무수정 통과**
- (b) Tailwind 등 동등 유틸 제공 → **클래스 1차 치환** (`border-l-4 border-{color}-400`)
- (c) 둘 다 없음 → **자체 강조 컴포넌트 1종 신설** (`<Callout color="red">…</Callout>` 등). 이 경우 색·여백·다크 모드 대비를 본 체크리스트의 (1.3) 합격선에 맞춰 정의

### 3.4 기타 보존 항목 (참고)

| 자산 | 검증 |
|------|------|
| 두 컬럼 grid (`<div grid="~ cols-2 gap-8">`) | "사전지식", "준비물 분담", "결과물 검증·문제 발생 시", "학생·일반인 — 꼭 지켜야 할 4가지" 슬라이드에서 동작 |
