## 2. N-1 시안 — 3축 카피 카드 (Why 도입부)

**3축 순서 추천**: 시간 → 생존 → 확장

> 추천 사유: 즉각 효용(시간) → 위기감(생존) → 해결책(확장) 순으로 청중 정서를 끌어올림. 마지막 "능력의 확장"이 곧이어 등장하는 N-2 사다리("그래서 어떻게?")와 자연스럽게 이어짐.

```markdown
### 왜 지금, 당신이 AI를 배워야 할까요?

<div class="grid" markdown>

!!! tip "① 시간 절약"
    **당신의 1시간은 얼마입니까?**

    매일 반복되는 30분짜리 작업, 하루 1시간이 모이면 1년에 250시간입니다.
    가장 소중한 자원은 시간입니다.

!!! warning "② 직업적 생존"
    **AI는 당신을 대체하지 않습니다.**
    **AI를 쓰는 사람이 당신을 대체합니다.**

    채용·평가·성과의 기준선이 빠르게 이동하고 있습니다.

!!! success "③ 능력의 확장"
    **이제 코딩을 몰라도, 자신의 일을 자동화할 수 있습니다.**

    대규모 시스템 개발이 아니라, **본인 업무·학습·일상에 쓰는 작은 자동화**가 비개발자에게 열렸습니다.

</div>
```

> 트랙 분리: 분리하지 않음(F-1 합의). 다만 ③의 부연 한 줄은 학생·일반인도 자연 포섭("업무·학습·일상").

## 3. N-2 시안 — 방법론 사다리 + 식당 비유

> ADR-0005 결과형 우회를 [3]단계에 자연 결합. W2의 "비개발자도 작은 소프트웨어 직접 만든다"를 [4] 수렴에 흡수(D-1 통폐합).

```markdown
### 그래서 어떻게? — 매번 시키기 vs 소프트웨어로 만들기

AI를 배운다고 해서 모두가 같은 방식으로 쓰는 것은 아닙니다.
**왜 이 강의가 "AI에게 매번 시키기"가 아니라 "AI로 소프트웨어 만들기"를 강조하는지** 한 단계씩 따라가 봅시다.

!!! abstract "① 출발은 누구나 같다"
    "AI에게 매번 새로 시키기"도 좋은 출발점입니다. 처음부터 본인의 챗봇이나 자동화 도구를 만드는 사람은 없습니다.

!!! note "② 그런데 우리 일은 대부분 '한 번'이 아니다"
    매주 반복되는 보고서, 매번 비슷한 회의록, 매학기 반복되는 학습 정리…
    **본인이 매일·매주 하는 일을 떠올려 보세요. 한 번으로 끝나는 일이 얼마나 됩니까?**

!!! warning "③ 반복인데 매번 처음부터 시키면 비용이 누적된다"
    - **매번 같은 지시를 다시 입력** (시간 누적)
    - **매번 결과가 조금씩 다름** (챗봇이 매번 답이 살짝 다른 그 느낌 — 품질 변동)
    - **매번 검수·수정** (이중 비용)

!!! success "④ 그래서 — 한 번 만들고 100번 쓰는 게 합리적"
    반복되는 일에는 **본인 일에 맞는 작은 소프트웨어를 직접 만드는 것**이 답입니다.
    이것이 본 강의가 약속하는 결과입니다 — **비개발자도 자기 일에 필요한 소프트웨어를 만든다.**

> 비유로 정리하면: **"매번 시키기"는 매번 식당에서 시켜 먹기**(편하지만 누적 비용↑·통제↓), **"소프트웨어로 만들기"는 본인 레시피로 자주 만드는 요리**(처음엔 품 들지만 본인 입맛에 딱·누적 효율↑). 본 강의는 **당신만의 레시피 1개**를 가져가는 것입니다.
```

**ADR-0005 정렬 점검**

| 항목 | 처리 |
|------|------|
| 비결정성 메커니즘 직접 노출 회피 | ✅ [3] ②번에 결과형 ("매번 결과가 조금씩 다름") |
| "한 번 만들면 동일한 품질로 반복 보장" | ✅ [4] 수렴 라인 + 식당 비유 ("본인 입맛에 딱") |
| "소프트웨어" 단어 정직 사용 | ✅ [4]에서 "본인 일에 맞는 작은 소프트웨어" 명시 — ADR-0005 핵심 못 ②에 해당 |
| "도구" 우회 표현 회피 | ✅ — 단, 식당 비유 안에서는 일상어 "도구" 한 번 등장 가능 (현재 시안엔 없음) |

## 4. W3 격상 시안 — 핵심 인용 ❌/✅ 박스화 (D-2 반영)

> N-2 직후, 본질 메시지를 ADR-0005 못 ②에 한 번 더 박는 자리.

**Before**

```markdown
> 핵심은 "AI에게 매번 시킨다"가 아니라, **"AI로 나만의 소프트웨어를 만들어 반복 자동화한다"** 는 것입니다.
```

**After (제안)**

```markdown
!!! quote "핵심: 비개발자도 소프트웨어를 만든다"
    ❌ "AI에게 매번 시킨다"

    ✅ **"AI로 나만의 소프트웨어를 만들어 반복 자동화한다"**
```

> 슬라이드(L150~)는 이미 같은 ❌/✅ 시각이라 정합 OK. 슬라이드 제목은 ADR-0005에 맞춰 "핵심: 비개발자도 소프트웨어를 만든다"로 정합.
