## 후속 이슈 후보 1 — 슬라이드 테마 재검토

- **현재 상태**: `@slidev/theme-seriph` 채택 (Issue #9 Task 1에서 `seriph` vs `apple-basic` 비교 후 선정)
- **선정 근거**: 빌트인 레이아웃(cover/two-cols/statement/quote/fact 등) 풍부, 한국어 Noto Serif 매칭 자연스러움
- **현 시점 미흡 메모**: 발표 환경에서 디자인 개선 여지 있음 (구체 항목은 새 이슈 spec 작성 시 사용자가 정리)
- **새 이슈 spec 작성 시 고려할 점**:
  - 후보 테마 비교(공식 테마 외 커뮤니티 테마 포함)·자체 커스터마이징 여부 결정
  - 디자인 변경 시 ADR-0001 메시지(3단계 모델)·ADR-0002 변형 금지 자산은 그대로 유지
  - 본 이슈에서 도입한 강조 박스(`border-l-4` UnoCSS) 패턴이 새 테마에서도 작동하는지 확인 — 안 되면 일관 강조 컴포넌트 신설 검토
  - 가능하면 별도 ADR로 디자인 결정 근거 기록 (테마 결정이 향후 슬라이드 작성 비용에 직결되므로)

## 후속 이슈 후보 2 — "보안 및 개인정보 가이드" 슬라이드 추가

- **현재 상태**:
  - `docs/security-guide.md` 존재 (MkDocs nav에 등록됨 — 문서 사이트에서는 정상 노출)
  - `slides/slides.md`에는 해당 섹션 **부재** (부수 언급 "실습용 가상 데이터(개인정보 미포함)" 1건뿐, L214)
- **공백의 의미**: Issue #9 spec은 `docs/index.md`의 Why/Who/What/How 흐름을 슬라이드로 이식하는 데 집중. `security-guide.md`는 별도 페이지라 본 마이그레이션 범위에 포함되지 않았음
- **새 이슈 spec 작성 시 고려할 점**:
  - 보안·개인정보 가이드를 본 슬라이드 흐름의 어느 지점에 삽입할지 (예: How 다음, 또는 별도 부록 섹션)

  - 콘텐츠 SSoT는 여전히 `docs/security-guide.md` (ADR-0002 단방향 동기화)
  - sync 체크리스트(README)에도 신규 섹션 점검 항목 추가 필요
