# ADR-0001: 실습 도구 범위(Claude 단독) 및 AI 활용 3단계 모델 도입

- **상태**: Accepted
- **결정일**: 2026-04-18
- **연관 이슈**: [Issue #7](https://github.com/scroogy-dev/ai-onboarding/issues/7)
- **영향 범위**: `docs/index.md`, `docs/security-guide.md`, `docs/operation-guide.md`, 추후 실습 페이지 설계

---

## 배경 (Context)

본 교육은 비개발자(임직원·학생·일반인) 대상 AI 활용 교육이다. 기존 설계는 Claude를 주축으로 하면서 학생 트랙 일부에서 Gemini Gems를 병행 사용하는 혼합 구조였다. 이로 인해 다음 문제가 관찰됐다.

1. **트랙 간 비대칭** — 임직원 트랙은 Claude만, 학생 트랙만 Gemini Gems 혼용 → 가이드·스크린샷·계정 준비물이 분기됨.
2. **준비물 복잡도** — 참가자가 Claude Pro 계정과 Gemini 계정 두 가지를 준비해야 할 수 있음.
3. **운영 부하** — 강사 1인당 10~15명 지원(운영가이드 기준)에서 도구 2종 트러블슈팅은 실질 병목.
4. **학습 사다리 설계 부재** — 참가자가 자신의 현재 AI 활용 수준을 자각하고 다음 단계로 나아갈 수 있는 **교육적 프레임**이 없었음. 실습이 "무엇을 쓰는가"(도구 중심)에만 치우쳐 있었음.

또한 기존에는 2단계(프롬프트 재활용)와 3단계(에이전트) 사이의 학습 연속성이 문서에 드러나지 않았다. 실제로는 2단계에서 익히는 **프롬프트 재활용·Agent Skills 개념**이 3단계 **Claude Cowork**에서 그대로 재활용 가능하여, 단일 생태계 내 학습 확장이 가능하다.

---

## 결정 (Decision)

### 1. 실습 도구 범위: **Claude 단독**

- 본 교육의 실습 도구는 **Claude(앤트로픽)** 으로 통일한다.
- Gemini·ChatGPT 등 다른 챗봇은 **1단계(챗봇 사용) 맥락에서 예시로만 언급**하며, 실습에는 사용하지 않는다.
- 기존 Gems 기반 학생 실습(AI 오답노트, 자동 출제기)은 **Claude의 자산화 기능(Agent Skills · Projects) 기반으로 재설계**한다 (상세 시나리오는 별도 이슈).

### 2. AI 활용 3단계 모델 도입

| 단계 | 명칭 | 내용 | 대표 도구·기능 | 본 교육에서의 다룸 |
|------|------|------|--------------|------------------|
| 1 | 챗봇 사용 | 단발성 대화로 답을 얻음 | Claude/ChatGPT/Gemini 웹 챗봇 | **참가 전제** (이미 경험) |
| 2 | 프롬프트 재활용 | 반복 사용 가능한 맞춤 챗봇·프롬프트 자산화 | **Agent Skills 기초**, Claude Projects | **2단계용 실습 진행** |
| 3 | 에이전트 활용 | 로컬 파일·작업을 자동화하는 에이전트 | **Claude Cowork**, Claude Code | **3단계용 실습 진행** |

**학습 사다리**: 2단계에서 익힌 프롬프트 재활용·Agent Skills 개념이 3단계 Claude Cowork에서 재활용되며, 동일 생태계 안에서 자연 확장된다.

### 3. 경계 명확화

- **Claude Code(CLI)**: 3단계 보조 도구로 거론한다. 본격 실습은 **Claude Cowork** 중심으로 진행하며, Claude Code는 비개발자에게 진입 장벽이 있어 옵션으로만 안내한다 (Claude Desktop 설치는 Cowork 사용을 위한 공통 준비물로 안내).
- **2단계·3단계 실습 시나리오 상세 설계**: 본 이슈(#7) 범위 외. **별도 이슈**에서 다룬다. 본 이슈에서는 실습의 존재·단계 매핑·문서 구조만 반영한다.

### 4. 대상자 재정의

- 학생 트랙 대상을 **"비개발자 학생·일반인"** 으로 명시 표기한다.
- **개발 진로 희망 학생은 본 교육 범위 외**. 별도의 개발 교육을 권장한다.

## 고려한 대안 (Alternatives considered)

### 대안 A. Claude + Gemini 병행 (기존 설계)

- **장점**: 도구 간 비교 학습 가능, Gemini 무료 티어 활용 가능(비용 장벽 완화), Google Workspace 연동 강점.
- **단점**: 학습 사다리가 끊김(Gems → Claude Cowork 연결 부자연), 인지 부하 ↑, 운영 부하 2배, 트랙 비대칭.
- **기각 사유**: 본 교육의 목표("1개라도 실제 반복해서 쓸 수 있는 결과물 만들기")는 도구 비교가 아니라 **산출물 완성률**에 달렸다. 2단계→3단계 학습 연속성이 산출물의 실제 업무 적용률을 끌어올리는 핵심 경로로 판단됨.

### 대안 B. Gemini 단독

- **장점**: 학생 접근성(무료 티어), Google Workspace 친화적.
- **단점**: 3단계(에이전트)의 Claude Cowork 수준 실습 대체 불가. 엔터프라이즈 환경에서 Claude 도입이 우세한 현재 추세와 맞지 않음.
- **기각 사유**: 3단계 학습 사다리를 구현할 수 없음.

### 대안 C. 단계 모델 없이 도구 중심 설계 유지

- **장점**: 구조 변경 불필요.
- **단점**: 참가자가 자신의 현재 수준과 다음 단계를 인지할 수 없어 교육 후 지속 학습이 이어지지 않음.
- **기각 사유**: 교육 효과의 지속성을 확보할 수 없음.

## 결과 (Consequences)

### 긍정적 영향

- **트랙 간 일관성**: 임직원·학생·일반인 모두 Claude Pro 1종으로 통일.
- **운영 부하 감소**: 가이드·스크린샷·트러블슈팅이 단일 도구 기준.
- **학습 사다리 명료화**: 참가자가 자신의 현 단계를 인지하고 다음 단계를 목표로 삼을 수 있음.
- **장기 확장성**: Agent Skills 같은 공통 개념이 2·3단계를 가로질러 재활용됨.

### 부정적 영향·리스크

- **비용 장벽**: 모든 참가자에게 Claude Pro(월 $20 수준)가 필요. 학생·일반인 트랙에 진입 장벽으로 작용 가능.
    - *완화*: 참가자 준비사항에 비용과 단서(변동 가능)를 명시하여 사전 인지시킴.
- **도구 종속**: Claude의 서비스 정책·가격 변동이 교육 전체에 영향.
    - *완화*: 보안 원칙은 도구 독립적으로 서술(security-guide 공통 원칙).
- **Gemini 장점 포기**: Google Workspace 연동·무료 티어 접근성은 본 교육에서 다루지 않음.
    - *완화*: 1단계 설명에서 다른 챗봇 선택지가 있다는 사실만 언급.

### 후속 조치

- `docs/index.md` Who·How 섹션에 3단계 모델과 대상자 재정의 반영 (Issue #7 Task 4).
- `docs/operation-guide.md` 사전설문 "현재 AI 활용 수준" 선택지를 3단계 모델에 맞춰 재정비 (Issue #7 Task 4).
- 2단계·3단계 실습 시나리오 상세 설계를 위한 **별도 이슈 생성**.
- Gems 기반 실습(오답노트·자동 출제기)의 Agent Skills · Projects 재설계를 후속 이슈에서 처리.
